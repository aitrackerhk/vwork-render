# 手機端 JavaScript 錯誤查看指南

## 功能說明

本系統已內建 JavaScript 錯誤處理器，可以在手機上直接查看和調試 JavaScript 錯誤，無需連接電腦或使用開發者工具。

## 功能特點

1. **自動捕獲錯誤**：自動捕獲所有 JavaScript 錯誤和未處理的 Promise 拒絕
2. **視覺化顯示**：在頁面底部顯示錯誤面板，方便查看
3. **詳細信息**：顯示錯誤訊息、來源位置、堆疊追蹤等
4. **手機優化**：專為手機螢幕設計，觸控友好
5. **錯誤歷史**：保存最近 50 個錯誤記錄

## 使用方法

### 自動顯示

當頁面發生 JavaScript 錯誤時，錯誤面板會自動出現在頁面底部，顯示錯誤信息。

### 手動控制

在瀏覽器控制台中可以使用以下命令：

```javascript
// 顯示錯誤面板
JSErrorHandler.show()

// 隱藏錯誤面板
JSErrorHandler.hide()

// 清除所有錯誤記錄
JSErrorHandler.clear()

// 獲取所有錯誤記錄
JSErrorHandler.getErrors()
```

### 面板操作

錯誤面板提供以下操作按鈕：

- **清除按鈕** (🗑️)：清除所有錯誤記錄
- **展開/收起按鈕** (▲/▼)：展開或收起錯誤列表
- **關閉按鈕** (✕)：隱藏錯誤面板（可通過 `JSErrorHandler.show()` 重新顯示）

### 查看錯誤詳情

每個錯誤項目包含：

- **錯誤訊息**：錯誤的主要描述
- **時間戳記**：錯誤發生的時間
- **來源位置**：錯誤發生的檔案和行號
- **堆疊追蹤**：點擊「顯示堆疊」可查看完整的錯誤堆疊

## 錯誤面板位置

- **手機端**：固定在頁面底部，佔據最多 50% 的視窗高度
- **桌面端**：固定在右下角，最大寬度 600px

## 注意事項

1. 錯誤面板預設會顯示，如果之前手動關閉過，需要透過 `JSErrorHandler.show()` 重新顯示
2. 錯誤記錄最多保存 50 個，超過會自動刪除最舊的記錄
3. 刷新頁面後錯誤記錄會清空
4. 錯誤面板使用深色主題，適合在各種環境下查看

## 技術細節

- 錯誤處理器使用全局 `error` 和 `unhandledrejection` 事件監聽器
- 所有錯誤同時會輸出到瀏覽器控制台（如果可用）
- 錯誤面板使用 Bootstrap Icons 顯示圖標
- 樣式完全自定義，不依賴外部 CSS 框架

## 故障排除

如果錯誤面板沒有顯示：

1. 檢查瀏覽器控制台是否有 JavaScript 錯誤（可能錯誤處理器本身載入失敗）
2. 確認 `error-handler.js` 已正確載入
3. 嘗試手動調用 `JSErrorHandler.show()`
4. 檢查是否有其他 JavaScript 錯誤阻止了錯誤處理器的初始化

## 開發建議

在開發環境中，建議：

1. 同時使用瀏覽器開發者工具和錯誤面板
2. 定期清除錯誤記錄，避免累積過多
3. 在生產環境中，可以考慮將錯誤發送到服務器進行分析（需要實現後端 API）
